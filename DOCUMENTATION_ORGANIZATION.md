# 📁 Documentation Organization

Struktur dokumentasi yang terorganisir untuk project Taman Kehati.

---

## 📂 Directory Structure

### Root Directory
Hanya berisi:
- ✅ `README.md` - Main project README
- ✅ `DOCUMENTATION_ORGANIZATION.md` - This file (organization guide)

### `docs/` - Active Documentation

#### `docs/client/` - Client Deployment Documentation
Dokumentasi khusus untuk client:
- `QUICK_START_CLIENT.md` - ⭐ START HERE - Quick start (5 steps)
- `CLIENT_DEPLOYMENT_GUIDE.md` - Complete deployment guide
- `DEPLOYMENT_CHECKLIST.md` - Pre & post checklist
- `CLIENT_DELIVERY_SUMMARY.md` - Delivery summary
- `FILES_TO_DELIVER.md` - Files list
- `FINAL_DELIVERY_CHECKLIST.md` - Final checklist
- `NEXT_STEPS_BEFORE_DELIVERY.md` - Next steps

#### `docs/migration/` - Migration Documentation
Database migration docs:
- `MIGRATION_STATUS.md` - Migration status
- `RAILWAY_DATABASE_VERIFICATION.md` - Railway verification
- `ALEMBIC_UPDATE.md` - Alembic update info

#### `docs/` (root of docs/)
General documentation:
- `DOCKER_SETUP.md` - Docker setup guide
- `DOCKER_QUICK_START.md` - Docker quick reference
- `README_DOCKER.md` - Docker README
- `CLEANUP_SUMMARY.md` - Cleanup summary (consolidated)
- `ORGANIZATION_SUMMARY.md` - File organization summary
- `DOCUMENTATION_INDEX.md` - Documentation index

Other subdirectories:
- `docs/getting-started/` - Getting started guides
- `docs/development/` - Development guides
- `docs/deployment/` - Deployment guides
- `docs/architecture/` - Architecture docs
- `docs/security/` - Security docs
- `docs/features/` - Feature docs
- `docs/troubleshooting/` - Troubleshooting
- `docs/runbooks/` - Operational runbooks
- `docs/reference/` - Reference docs

### `archive/` - Historical Documentation
Organized by category:
- `archive/deployment/` - Old deployment docs
- `archive/features/` - Feature implementation history
- `archive/fixes/` - Bug fix history
- `archive/sop/` - Standard Operating Procedures
- `archive/analysis/` - Analysis documents
